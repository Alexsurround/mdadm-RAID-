# Оптимизация дисковой подсистемы RAID 5 и приоритетов процессов (Debian 10)

Репозиторий содержит наработки и конфигурационные файлы для устранения фризов видеозаписи (`avregd`) и зависания виртуальных машин `VirtualBox` во время активного копирования файлов. 

Главная проблема заключалась в высоком значении `%iowait` (до 65%), из-за чего RAID 5 массив полностью блокировал случайную запись чувствительных к задержкам сервисов.

## 🛠 Выполненные настройки

### 1. Настройка планировщика ввода-вывода (BFQ)
Для всех физических дисков массива (`sda`, `sdb`, `sdc`) активирован планировщик **BFQ** (Budget Fair Queueing), который распределяет ресурсы диска честно между процессами и предотвращает монополизацию канала утилитами копирования.

Постоянное правило udev сохранено в `/etc/udev/rules.d/60-scheduler.rules`:
```text
ACTION=="add|change", KERNEL=="sd[a-z]", ATTR{queue/scheduler}="bfq"
ACTION=="add|change", KERNEL=="md*", ATTR{queue/scheduler}="none"
```

### 2. Оптимизация кэша грязных страниц (Память ядра)
Изменены лимиты сброса дискового кэша в `/etc/sysctl.conf`. Это заставляет Linux сбрасывать данные на диск маленькими порциями (15-50 МБ), не накапливая гигабайты в RAM и исключая резкие "затыки" системы:
```text
vm.dirty_background_bytes = 15728640
vm.dirty_bytes = 52428800
```

### 3. Управление приоритетами (Nice / Ionice)
* **Приоритет CPU**: Для виртуальных машин настроен запуск с повышенным приоритетом (`nice -n -10`) через конфигурацию `limits.conf`, что выделяет им больше процессорного времени.
* **Приоритет диска (I/O)**: Скрипты копирования переведены в класс `Idle` (низший дисковый приоритет) с жестким ограничением скорости.

## 🚀 Как запускать тяжелое копирование теперь

Чтобы `%iowait` не поднимался выше безопасных 15-18%, любое копирование больших файлов/бэкапов должно выполняться с искусственным ограничением скорости (Throttling) до **20–25 МБ/с**.

**Вариант с `rsync`:**
```bash
ionice -c 3 rsync --bwlimit=25000 -avP /откуда /куда
```

**Вариант с `cp` (требуется пакет `pv`):**
```bash
tar -cf - -C /откуда . | ionice -c 3 pv -L 25m | tar -xf - -C /куда
```

## 📊 Результаты оптимизации
* Средний показатель `%iowait` при пиковых нагрузках упал с **28.57%** до **17.87%**.
* Полностью исчезли пиковые задержки в 60%+.
* Потоковая видеозапись `avregd` и гостевые ОС в VirtualBox работают плавно, без потери кадров и фризов.
