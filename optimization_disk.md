# Оптимизация дисковой подсистемы RAID 5 и приоритетов процессов (Debian 10)

Репозиторий содержит конфигурации, скрипты и документацию для устранения фризов видеозаписи (`avregd`) и зависания виртуальных машин `VirtualBox` во время тяжелого копирования файлов. 

Главная проблема заключалась в высоком значении `%iowait` (в пиках до 65%), из-за чего RAID 5 массив полностью блокировал случайную запись чувствительных к задержкам сервисов. Настройки процессора (CPU) не помогали, так как процессор простаивал (`%idle` > 60%), ожидая ответа от дисков.

---

## 🛠 Выполненные настройки и диагностика

### 1. Переключение планировщика ввода-вывода на BFQ
Для предотвращения монополизации диска утилитами копирования был выбран планировщик **BFQ** (Budget Fair Queueing). Он распределяет ресурсы диска честно между процессами и гарантирует, что `avregd` получит свой квант времени на запись без задержек.

#### Тестирование и применение «на лету»:
Чтобы не перезагружать сервер и сразу проверить работу планировщика, его активация для каждого диска массива (`sda`, `sdb`, `sdc`) выполняется через перенаправление потока в `sysfs`:

```bash
# Включение BFQ для диска sda
echo "bfq" | sudo tee /sys/block/sda/queue/scheduler

# Включение BFQ для остальных дисков массива
echo "bfq" | sudo tee /sys/block/sdb/queue/scheduler
echo "bfq" | sudo tee /sys/block/sdc/queue/scheduler
```

#### Проверка текущего статуса:
Убедиться, что планировщик применился (активный алгоритм должен быть в квадратных скобках `[bfq]`):
```bash
grep . /sys/block/sd*/queue/scheduler
```
*Вывод должен быть следующим:*
```text
/sys/block/sda/queue/scheduler:mq-deadline [bfq] none
/sys/block/sdb/queue/scheduler:mq-deadline [bfq] none
/sys/block/sdc/queue/scheduler:mq-deadline [bfq] none
```

#### Постоянное сохранение при перезагрузке (Udev):
Чтобы настройки не сбрасывались после перезапуска системы, создан файл правил `/etc/udev/rules.d/60-scheduler.rules`:
```text
# Включаем BFQ для всех физических дисков (sda, sdb, sdc...)
ACTION=="add|change", KERNEL=="sd[a-z]", ATTR{queue/scheduler}="bfq"

# Отключаем планировщик для виртуального RAID-массива (ускоряет обработку ядром)
ACTION=="add|change", KERNEL=="md*", ATTR{queue/scheduler}="none"
```
Принудительно применить правила udev без перезагрузки: `sudo udevadm trigger --action=change`

### 2. Оптимизация кэша грязных страниц (Память ядра)
По умолчанию Linux агрессивно кэширует запись в RAM, что приводило к накоплению гигабайтов данных и резким «затыкам» диска при их сбросе. В файл `/etc/sysctl.conf` внесены лимиты, заставляющие ядро сбрасывать кэш мелкими порциями (15–50 МБ). 

**Важно:** В файле не должно быть строчных комментариев (знаков `#`) в конце строк с параметрами!

```text
vm.dirty_background_bytes = 15728640
vm.dirty_bytes = 52428800
```
Применить настройки ядра немедленно: `sudo sysctl -p`

---

### 3. Управление приоритетами (Nice / Ionice)
* **Приоритет CPU**: Для стабильной работы виртуальных машин VirtualBox настроен запуск с повышенным приоритетом (`nice -n -10`) через `/etc/security/limits.conf`.
* **Приоритет диска (I/O)**: Процессы копирования принудительно переводятся в класс `Idle` (низший дисковый приоритет) с жестким искусственным ограничением скорости (Throttling).

## 🚀 Как запускать тяжелое копирование файлов теперь

Чтобы `%iowait` не поднимался выше безопасных 15-18%, любое копирование больших файлов или бэкапов на RAID 5 должно идти на скорости **не более 20–25 МБ/с**.

**Вариант А. Через `rsync` (с ограничением полосы в КБ/с):**
```bash
ionice -c 3 rsync --bwlimit=25000 -avP /откуда /куда
```

**Вариант Б. Через `cp` (с использованием утилиты `pv` для урезания скорости потока):**
```bash
# Требуется пакет: sudo apt install pv
tar -cf - -C /откуда . | ionice -c 3 pv -L 25m | tar -xf - -C /куда
```

## 📊 Результаты оптимизации
Контроль через утилиту `sar -u 1 60` показал:
* Средний показатель `%iowait` при пиковых нагрузках упал с **28.57%** до **17.87%**.
* Полностью исчезли критические пики задержек в 55-65%.
* Потоковая видеозапись `avregd` и гостевые ОС в VirtualBox работают плавно, без потери кадров и фризов.
